**API Design:**

```csharp
// Connection string
var connStr = "Data Source=db.witdb;ParallelMode=Auto|WAL|Partitioned|None";

// Builder
var db = new WitDatabaseBuilder()
    .WithFilePath("db.witdb")
    .WithBTree()
    .WithParallelWrites(ParallelMode.Auto)  // New
    .WithParallelWriters(4)                 // Optional: num parallel writers
    .Build();

// Auto mode selection
public enum ParallelMode
{
    None,        // Single-threaded (current behavior)
    Auto,        // Detect best mode
    WAL,         // WAL-based parallelism
    Partitioned  // Partition-based (LSM only)
}
```

## Performance Targets

| Metric | Current | Target (Parallel) | Improvement |
|--------|---------|-------------------|-------------|
| Insert 10K rows (1 thread) | ~130ms | ~130ms | Same |
| Insert 10K rows (4 threads) | ~130ms* | ~50ms | **2.6x** |
| Insert 10K rows (8 threads) | ~130ms* | ~35ms | **3.7x** |
| Mixed read/write (4 threads) | ~200ms | ~80ms | **2.5x** |

*Currently parallel = sequential due to global lock

## Decision Log

| Date | Decision | Rationale |
|------|----------|-----------|
| 2025-06-11 | Start with WAL-based approach | Simpler, proven pattern, ACID guarantees |
| 2025-06-11 | Make parallel mode opt-in | Don't break existing behavior |
| 2025-06-11 | Use System.Threading.Channels | Lock-free, high performance, built-in backpressure |
